# Architecture Decision Records

This directory contains Architecture Decision Records (ADRs) — short documents that capture important architectural decisions, their context, and consequences.

## What is an ADR?

An Architecture Decision Record is a short text file documenting a significant architectural decision. Each ADR:

- Describes the **context** and **problem** we faced
- Lists the **options** considered
 States the **decision** we made
- Notes the **consequences** of that decision

## How to Use

When making a significant architectural decision:

1. Copy `ADR-001-record-architecture-decisions.md` as a template
2. Give it the next sequential number and a descriptive title
3. Fill in the sections
4. Commit alongside the code that implements the decision
